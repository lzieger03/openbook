LERNSTANDMODELL - openbook/learning
=====================================

ZIEL:
Die KI soll wissen wer auf welchem Stand ist und welche Fortschritte ein
Nutzer macht. Der ChatConsumer liest diese Daten beim Verbindungsaufbau
und gibt sie der KI als Kontext mit.


WAS GEBAUT WERDEN MUSS:
------------------------

1. models/state.py  →  LearningState
   - Welcher User ist in welchem Kurs eingeschrieben
   - Welche TextbookPage hat er zuletzt gelesen
   - Gesamtfortschritt in Prozent
   Felder:
     user        → ForeignKey(settings.AUTH_USER_MODEL, CASCADE)
     course      → ForeignKey(openbook_content.Course, CASCADE)
     last_page   → ForeignKey(openbook_content.TextbookPage, SET_NULL, null=True)
     progress    → FloatField(default=0.0)  # 0.0 bis 1.0
     updated_at  → DateTimeField(auto_now=True)

2. models/objective.py  →  LearningObjective
   - Definiert ein Lernziel innerhalb eines Kurses (z.B. "Kapitel 3 verstehen")
   - Kann vom Kursersteller angelegt werden
   Felder:
     course      → ForeignKey(openbook_content.Course, CASCADE)
     name        → CharField
     description → TextField
     page        → ForeignKey(openbook_content.TextbookPage, SET_NULL, null=True)

3. models/quiz_result.py  →  QuizResult
   - Speichert jeden Quiz-Versuch eines Nutzers
   - Wichtig damit die KI schwache Stellen erkennt
   Felder:
     user        → ForeignKey(settings.AUTH_USER_MODEL, CASCADE)
     page        → ForeignKey(openbook_content.TextbookPage, CASCADE)
     score       → FloatField()   # 0.0 bis 1.0
     attempts    → PositiveIntegerField(default=1)
     answered_at → DateTimeField(auto_now_add=True)


DANACH (Serializers + Viewsets):
---------------------------------
Für jedes Modell gibt es bereits leere Dateien:
  serializers/state.py, objective.py, quiz_result.py
  viewsets/state.py, objective.py, quiz_result.py

Routen werden in routes.py registriert (Kommentare dort als Vorlage).
Schau dir openbook/content/ als Referenz an wie Serializer und Viewsets
im Projekt aufgebaut werden.


ANBINDUNG AN DIE KI (letzter Schritt):
----------------------------------------
In openbook/ai/consumers/chat.py, in der __init__()-Methode:
  - LearningState des eingeloggten Users laden
  - QuizResults der letzten X Versuche laden
  - Als Kontext-String zusammenbauen und beim ersten LLM-Call mitsenden

Beispiel Kontext-String fuer das LLM:
  "Der Nutzer ist in Kurs 'Mathe Grundlagen' eingeschrieben.
   Fortschritt: 42%. Zuletzt gelesen: Seite 'Bruchrechnung'.
   Letztes Quiz: Score 0.3 (schlecht) auf Seite 'Division'."


QUIZ-GENERIERUNG:
-----------------
Quiz-Fragen werden dynamisch ueber den bestehenden kursbezogenen Chat-WebSocket
generiert. Das statische KI-Pair-Quiz mit JSON-Datei/QuizQuestion-Modell wurde
entfernt.

Aktuelles Verhalten:
  - Frontend sendet quiz_start ueber /ws/ai/courses/<course_id>/chat
  - AssistantOrchestrator generiert Fragen zuerst aus RAG-Dokumenten
  - Ohne RAG-Dokumente wird Kurskontext aus Kursdaten, Skills und TextbookPages
    verwendet
  - Backend liefert quiz_generated mit Multiple-Choice-Fragen zurueck


MODELLE ALS IMPORTS FREISCHALTEN:
-----------------------------------
Nach der Implementierung in models/__init__.py einkommentieren:
  from .objective  import LearningObjective
  from .state      import LearningState
  from .quiz_result import QuizResult

Dann Migration erstellen:
  python manage.py makemigrations openbook_learning
  python manage.py migrate
