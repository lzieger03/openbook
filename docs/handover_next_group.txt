Project Overview
----------------
ELISA is the student project name for the OpenBook learning assistant proof of concept. It extends
OpenBook with course-aware learning support: assistant chat, RAG-based course context, quiz flows,
learning progress, teacher and student frontend areas, and first gamification features.

The work is not a finished production system. Several branches are integration branches that combine
work from multiple students. Some files are implementation code, some are generated frontend or API
artifacts, and some are planning or handover notes. Treat the current state as a proof of concept
that needs cleanup, migration checks, and product decisions before a final release.

Important Commands
------------------
Use Poetry for Python commands. On this Windows setup, Poetry is available through py -m poetry.

Check Django in the Poetry environment:

    py -m poetry run python -c "import django; print(django.get_version())"

Start the Django development server from the repository root through npm:

    py -m poetry run npm run runserver

Run backend tests from the existing npm script:

    py -m poetry run npm run test:backend

Build frontend packages:

    npm run build

Run the full docs command:

    $env:Path += ";C:\Program Files\Graphviz\bin"
    py -m poetry run npm run docs

Run a one-shot Sphinx build without OpenAPI synchronization:

    $env:Path += ";C:\Program Files\Graphviz\bin"
    py -m poetry run sphinx-build -T -v docs/ docs/_build

Run Sphinx autobuild without OpenAPI synchronization:

    $env:Path += ";C:\Program Files\Graphviz\bin"
    py -m poetry run sphinx-autobuild -T -v --port 8885 docs/ docs/_build

Student Code Overview
---------------------
The student code is distributed across backend, frontend, and documentation areas.

Assistant and AI:
- src/openbook/assistant/ adds an assistant app with document model, admin integration, management
  command, RAG client, vector index, prompt builder, orchestration, learning context, viewsets, and
  tests.
- src/openbook/ai/ was changed for chat consumer and message handling integration.
- RAG-related commits synchronize or use course textbooks and documents as assistant context.

Learning progress:
- src/openbook/learning/ adds LearningState, QuizResult, serializers, viewsets, routes, signals,
  tests, README, and TODO notes.
- The documented purpose is to track last page, completed pages, quiz scores, and attempts so the AI
  assistant can receive learning context.

Quiz:
- The current documented quiz direction uses the course chat WebSocket. The frontend sends quiz_start
  and receives generated multiple-choice questions.
- Later work adds server-side quiz grading.
- An earlier src/openbook/quiz/ proof-of-concept appears in one integration branch. Treat it as
  legacy or provisional until the team decides the final quiz architecture.

Gamification:
- src/openbook/gamification/ adds models, admin classes, routes, viewsets, services, signals,
  migrations, management commands, and tests.
- Concepts include rewards, reward event logs, account progress, streaks, levels, skills, skill
  progress, and course progress.
- Learning activity can update streaks and eventually award rewards, but exact reward rules remain
  partly open.

Frontend:
- src/frontend/dashboard/ adds a student dashboard proof of concept with dashboard page, content
  page, quiz page, course chat page, profile edit page, chat widget, panels, stores, API wrappers,
  and styling.
- src/frontend/teacher/ adds a teacher frontend proof of concept with course list, course detail,
  content panel, overview panel, students panel, store, API wrappers, and styling.
- src/frontend/app/ includes integration changes such as routes, auth store changes, gamification
  cards, sections, and generated API client artifacts in some branches.

Known Problems
--------------
The following problems were observed or documented:

- The full npm docs command can fail during docs:sync-openapi when the local database is not
  migrated. The observed error was: no such table: django_site.
- When OpenAPI synchronization fails, docs/_openapi files can be missing. A pure Sphinx build then
  succeeds but reports download warnings for missing OpenAPI files.
- Graphviz must be available on PATH for graphviz-based documentation rendering on Windows.
- Some integration branches include generated frontend dist files and duplicate-looking files such as
  names containing " 2". These should be reviewed before cleanup or merge.
- The exact rules for quiz points, course completion rewards, and long-term learning memory entries
  are not final.
- The teacher reporting workflow is not documented as complete.
- AI-generated answers and quizzes need validation before educational or assessment use.

Notes For Continuation
----------------------
Do not treat this handover as a final architecture decision. It is a documented snapshot of student
work visible in the repository. Before continuing implementation, inspect the current branch state,
run tests, run migrations, and decide which integration branch is the source of truth.

Avoid mixing project-owner maintenance branches into the student documentation. If a commit or branch
cannot be assigned to the student team, document it as unclear instead of guessing.
